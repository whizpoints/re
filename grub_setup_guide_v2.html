📌 Overview & Diagnostic State

This guide maps out the clean migration path from the standard Pop!_OS systemd-boot setup to a highly custom, graphical GRUB2 Bootloader Environment. This provides flawless disk discovery and unified dual-boot options.

💡 System Architecture Discovery Info
Pop!_OS Native EFI Target: Located at /dev/sda5 mapped to /boot/efi
Windows Partition Path: Located at /dev/sda1 containing Microsoft/Boot/bootmgfw.efi

🚀 Phase 1: Installation & Configuration Overwrite

Execute these core steps from a fresh terminal sequence to fully implement the clean GRUB architecture layout.

1. Install Core UEFI Platform Files

Fetch packages directly from the main OS repository nodes:

sudo apt update && sudo apt install grub-efi-amd64 grub-pc-bin git -y

2. Deploy GRUB Architecture to EFI Partition

Force build standard directories directly into your system's dedicated boot loader sector:

sudo grub-install --target=x86_64-efi --efi-directory=/boot/efi --bootloader-id=GRUB

3. Enable Dynamics OS-Prober Engine

Override security lockouts to dynamically capture and expose Windows installation parameters:

echo "GRUB_DISABLE_OS_PROBER=false" | sudo tee -a /etc/default/grub

4. Set Visual Timeout Window

Expose hidden menu styles and lengthen selection availability options to exactly 5 seconds:

sudo sed -i 's/GRUB_TIMEOUT_STYLE=hidden/GRUB_TIMEOUT_STYLE=menu/' /etc/default/grub
sudo sed -i 's/GRUB_TIMEOUT=0/GRUB_TIMEOUT=5/' /etc/default/grub

🎨 Phase 2: Ultra Clean UI Customization Themes

With the structural parameters set, apply premium high-resolution themes to replace low-res legacy interface assets.

git clone https://github.com/vinceliuice/grub2-themes.git
cd grub2-themes
sudo ./install.sh -t tela -s 1080p

Compile Changes & Build Active Selection Grid

Always run a compilation refresh to lock parameters securely into boot structures before power cycling:

sudo update-grub
⚠️ Critical Post-Installation Requirement
During system restart, initialize firmware configurations (Press F2 / Delete key). Navigate to the Boot Priority Order panel, locate option index #4, and shift it to **Position #1** to guarantee automated menu initialization.
